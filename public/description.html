Application Summary

This application serves as a maintainer validation system for Bioconductor. The application will send periodic validation emails to remind maintainers of Bioconductor policies. Maintainers must accept Bioconductor policies once a year to remain in good standing. Those failing to do so may have their packages deprecated/removed from Bioconductor. The application uses the emails listed in the package DESCRIPTION. Maintainers are required to keep this email up-to-date with an email that the Bioconductor Core Team can reach out to. If an email is determined to be unreachable, invallid, bounces, etc. it will also be flagged as potential deprecation/removal.

API Endpoint Summary

All metadata or lists returned as json

Route Method Description
/ GET Shows this API summary
/info/package/:pkg GET Get metadata by package name
/info/name/:name GET Get metadata by maintainer name (supports special characters)
/info/email/:email GET Get metadata by maintainer email
/info/valid/:email GET Check if an email is valid
/list/invalid/ GET List all invalid emails
/list/needs-consent/ GET List all emails needing consent
/list/bademails/ GET List all emails flagged by either invalid or needing consent
